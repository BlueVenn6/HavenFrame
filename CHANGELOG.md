# Changelog

## 0.2.0-rc.12

- 冻结中英双语、无本地部署产品边界；桌面内嵌 sidecar 仅作为应用基础设施。
- 保持 OpenAI `gpt-image-2` 原生/中转与 Google Gemini 图片生成线路，GLM 提取与生图配置独立。
- 修复 A4 方案板报告长文本越界、截断和实现细节暴露问题。
- 增加唯一 installer manifest、安装版 sidecar 生命周期验证与公开源码安全门禁。
- 原创源码采用 `AGPL-3.0-or-later`，第三方组件保留各自许可证。

## 0.1.0-rc.5

- 修复 OpenAI 图片中转未提供 `/models` 查询接口时被误报为“模型不支持”的问题；已有真实成功任务和落盘图片可作为线路验证证据。
- 修复平面图等图片工作流在当前输入为空时错误显示项目历史输出的问题，历史结果必须匹配当前输入资产。
- 增加工作流历史输入隔离的可执行前端回归测试。

## 0.1.0-rc.4

- Tauri 启动 sidecar 时传入桌面父进程 PID；GUI 正常退出、崩溃或被强制结束后 sidecar 都会自动停止。
- Windows 安装验证新增强制结束 GUI 的回归门禁，禁止遗留占用 `8000` 的孤立 sidecar。

## 0.1.0-rc.3

- 修复安装版中转 Base URL 与 API Key 保存后未可靠进入 AppData 的问题，并在保存响应后校验持久化结果。
- 增加打包 sidecar 的写入、模块绑定、重启读取发布门禁；门禁不调用外部 Provider。
- 桌面壳和前端校验后端服务身份与 API 协议版本，拒绝误连 `8000` 上的旧服务或其它服务。
- 从注册表和旧数据库迁移中删除三个不可运行的占位 Provider 配置。
- Android/iOS 共享模型设置保存同样校验中转地址与 Key 安全存储状态；移动端仍不包含本地部署能力。

## 0.1.0-rc.2

- 升级为普通用户安装包内测候选版。
- Windows 安装包内置 FastAPI PyInstaller sidecar，启动应用时自动拉起本地后端。
- 安装版默认写入 `%LOCALAPPDATA%\com.qigou.desktop\`，不写入安装目录。
- 保留本地 token、Host/Origin/CORS、URL Policy、数据流确认、上传限制、队列限制和 secret scan。
- 开发环境可通过显式测试开关验证本地任务队列、PNG 归档和日志；正式包不暴露测试渲染器。
- 增加端口 `8000` 被占用时的降级提示和排查命令。
- 当前 RC 安装包未代码签名，正式公开发布前需要签名。

## 0.1.0-rc.1

- 初始 RC：完成真实任务队列、归档、日志和发布前安全硬化，并隔离开发测试渲染器。
