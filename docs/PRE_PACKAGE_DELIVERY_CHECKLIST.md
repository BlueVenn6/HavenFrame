# HavenFrame / 栖构打包前完整交付验收清单

目标：在下一次 Windows 或 Android 打包前，一次性发现功能、语言、模型、报告、数据和发布链问题。除非本清单的“打包前门禁”全部通过，否则不生成新的安装包。

## 1. 使用规则

每项只能记录以下状态：

- `PASS`：按清单真实操作并得到正确结果。
- `FAIL`：真实操作失败，必须先修复并重新验证受影响范围。
- `BLOCKED_BY_CREDENTIALS`：缺少真实 Provider 凭据，不能用历史结果或 mock 代替。
- `NOT_RUN`：尚未执行，不能视为通过。
- `NOT_APPLICABLE`：产品明确不支持，并已说明原因。

打包前允许存在 `BLOCKED_BY_CREDENTIALS` 的前提是：该 Provider 不属于本轮必须交付的已配置线路，并且用户明确接受。任何 `FAIL` 或关键 `NOT_RUN` 都会阻止打包。

测试过程中先集中记录问题，不要发现一个问题就立即打包。完成一整轮后按 `P0 -> P1 -> P2 -> P3` 批量修复；所有修复完成后，使用同一个候选源码快照重新执行受影响项和完整冒烟测试。

## 5. 模型设置与真实 Provider

连接测试只能证明对应请求的结果，不能用“可访问”冒充真实生成。至少对本轮实际交付的线路各执行一次真实业务任务。

### 图片生成

- [ ] OpenAI Native：`gpt-image-2` 保存、测试、应用和真实生图一致。
- [ ] OpenAI-compatible Relay：Base URL、endpoint、Key、`gpt-image-2`、超时和真实生图一致。
- [ ] Google Gemini：所选 Gemini 图片模型、Key、配额和真实生图一致。
- [ ] 任务快照中的 provider、model、config ID、endpoint 类型与 UI 选择一致。
- [ ] 图片生成超时策略允许长任务完成，不回退到错误的 120 秒旧配置。
- [ ] 不存在不可运行的占位线路或失效中转地址。

### 多模态提取

- [ ] 中国大陆智谱 GLM 使用 `https://open.bigmodel.cn/api/paas/v4` 和对应大陆 Key。
- [ ] 国际 Z.AI GLM 使用 `https://api.z.ai/api/paas/v4` 和对应国际 Key。
- [ ] OpenAI-compatible 视觉中转仅在用户明确配置时出现。
- [ ] GLM 路线与图片生成路线完全独立，互不覆盖。
- [ ] 使用真实室内图片提取家具、材质、颜色和房间信息，结果写入当前项目。
- [ ] VPN、代理或 DNS 影响线路时，错误明确归类为网络/DNS，而不是模型不支持。

## 7. 各工作流人工验收

每个工作流都必须独立执行。GLM 提取、人工确认、预算、采购信息和图片生成之间，除非当前操作确有技术依赖，否则不得互相阻塞。

### 平面图

- [ ] 上传平面图、草图或黑白图后能够预览、替换和删除。
- [ ] 2D 彩色平面图和 3D 鸟瞰图分别可选并进入真实任务队列。
- [ ] 房间、风格、比例、提示词和模型选择均进入任务快照。
- [ ] 使用真实图片线路生成一张结果，图片可打开且内容不是历史占位图。
- [ ] 失败、取消、超时和重试状态正确。

### 单房间方案板

- [ ] 上传一张房间图后能够预览、替换和删除。
- [ ] 不做 GLM 提取、不填预算、不人工确认时，仍能直接生成方案板图片。
- [ ] 可单独执行 GLM 多模态提取，并逐项保留或删除。
- [ ] 预算、数量、采购状态、购买方式和链接均为可选，不阻塞其他操作。
- [ ] 可单独生成报告内容。
- [ ] 可单独生成方案板图片。
- [ ] 可单独导出 A4 图片报告和结构化表格。
- [ ] 页面刷新、应用重启和历史回看后结果仍存在。

### 多房间方案板

- [ ] 可上传、替换和删除多张不同房间图片。

- [ ] 不做 GLM、不填预算、不人工确认时，仍能直接生成整体或分房间方案板。
